# 宏曦标书 MVP 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 构建宏曦标书 MVP -- 用户上传招标文件(.docx/.pdf)，AI 自动解析需求并生成完整标书，人工审核修订后导出 Word + PDF。

**Architecture:** React 18 + Vite + FastAPI + PostgreSQL + Chroma + DeepSeek API, Docker Compose 部署阿里云。

**Tech Stack:** React 18, Vite, Ant Design 5, Tiptap, FastAPI, SQLAlchemy 2.0, Alembic, PostgreSQL 15, Chroma, python-docx, pdfplumber, DeepSeek API, Docker Compose, Nginx

## Global Constraints

- 系统名称: 宏曦标书 | 版权: 云南宏曦科技有限公司
- AI 主力: DeepSeek (适配器模式可切换)
- 部署: 阿里云 4核8G+ Docker Compose
- 初期用户: 2-3人
- 标书排版: 正文宋体小四 1.5倍行距, 标题黑体, 页边距国标
- 敏感数据脱敏后再入AI Prompt
- 所有外部依赖环境变量注入

### Task 4: 工具层 -- JWT 安全 & 数据脱敏

**Files:**
- Create: `backend/app/utils/__init__.py`
- Create: `backend/app/utils/security.py`
- Create: `backend/app/services/__init__.py`
- Create: `backend/app/services/deid.py`

**Produces:** `create_access_token()`, `get_password_hash()`, `verify_password()`, `get_current_user()` (Depends), `deidentify_text()`, `reidentify_docx()`

**security.py:** passlib bcrypt context. `create_access_token` returns JWT with sub=user_id and exp. `get_current_user` FastAPI dependency extracts user from token.

**deid.py:** `deidentify_text(text)` -> (deidentified_text, mapping_dict). Regex-based: replaces Chinese names (2-4 chars after specific context chars), ID numbers (18 digits), phone numbers (11 digits), certificate numbers. `reidentify_docx(text, mapping)` restores placeholders.

**Core design rule:** Before any content enters the AI prompt, call `deidentify_text()`. After AI returns content and before rendering to docx, call `reidentify_docx()`.

### Task 11: AI Pipeline -- 招标解析 & 标书生成

**File:** `backend/app/services/ai_pipeline.py`

**Produces:** `parse_bid_requirements(document_text) -> dict`, `generate_outline(requirements) -> list[dict]`, `generate_chapter(chapter_title, requirements, context) -> AsyncIterator[str]`, `generate_chapter_with_materials(...) -> AsyncIterator[str]`

**SYSTEM_PROMPT:** 定义AI角色为宏曦标书助手，包含5条写作规范（行业地道表达、具体事实锚点、禁止空泛词、关键词回应不照搬、句式多样化），版权声明。

**parse_bid_requirements:** 发送招标文件全文给AI，要求JSON格式返回：project_name, project_budget, project_duration, qualification_requirements[], personnel_requirements, service_requirements[], evaluation_criteria, special_requirements[], bid_sections[]

**generate_chapter_with_materials:** 组装context字符串（可用资质列表 + 匹配人员信息 + 相似历史章节），调用逐章生成。

### Task 12: Word 排版渲染引擎

**File:** `backend/app/services/render_engine.py`

**Produces:** `render_bid_to_docx(chapters, project_name, style_config) -> str` (output path), `export_to_pdf(docx_path) -> str`

**排版规范 (DEFAULT_STYLE):** 正文宋体12pt/1.5倍行距, 一级标题黑体16pt, 二级标题黑体14pt, 页边距上下2.54cm左右3.17cm, 页眉"云南宏曦科技有限公司", 页脚页码

**封面:** 项目名称居中(黑体22pt) + "投标文件"(18pt) + 公司名称 + 日期

**export_to_pdf:** 调用 LibreOffice headless 转换 (最佳效果)

### Task 13: AI 生成 API & SSE 流式推送

**File:** `backend/app/api/bid.py`

**Produces:** 4个核心接口

- **POST /upload-and-parse:** Multipart文件上传 -> 保存 -> 解析文档 -> AI提取需求 -> 创建BidProject -> 返回解析结果
- **POST /generate:** 接收project_id -> 生成outline -> 创建ProjectChapter记录 -> SSE逐章流式返回 -> 生成完成标记
- **POST /export:** 接收project_id + format -> 调用render_bid_to_docx -> 可选export_to_pdf -> 返回下载URL
- **GET /download/{filename}:** FileResponse 返回文件

**SSE事件类型:** `status` (当前进度), `chunk` (文本片段含chapter_id), `chapter_done` (章节完成), `done` (全部完成)
